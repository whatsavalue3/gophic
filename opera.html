Opera

Opera is an "expression" based programming language.

Key points:

  • The only thing that Opera defines are operators and expressions.
  • Opera can only reserve the standard ASCII graphic characters which are not letters, numbers or underscores.

There are several types of operators in Opera:

  • Prefix: Appears on the left side of an expression or multiple expressions.
  • Infix: Appears between two expressions, combining them.
  • Postfix: Appears on the right side of an expression.
  • Ternary: Joins three expressions with two different symbols separating them.

An expression is either a literal value or the result of an operation on expressions.

Addition, subtraction, multiplication, division, use the infix operators EXPR + EXPR , EXPR - EXPR , EXPR * EXPR , EXPR / EXPR respectively.

To remove any ambiguities from the order in which operations take place, you can use ( EXPR ) to contain an expression.

Negation is a prefix operation denoted as - EXPR

A number literal can be expressed in a couple of ways:

  • Decimal: If not 0, has to start with a digits different than 0.
  • Binary: Starts with 0b and continues with zeros and ones.
  • Octal: Starts with 0 and continues with digits 0 through 7.
  • Hexadecimal: Starts with 0x and continues with digits 0 through 9, and then a through f (case insensitive).

A number literal can use a . as a separator between the whole part and fractional part.

Opera supports a number of primary data types:

  • Integer: Denoted as # EXPR where the expression defines the number of bits of the integer.
  • Fixed point: Denoted as . EXPR EXPR where the first expression defines the number of bits of the fixed point number, and the second expression defines the number of bits after the point there are.
  • Floating point: Denoted as % EXPR where the expression defines the number of bits of the floating point value.

Opera defines the concept of a "place" to be an area in the code where the result of an expression resides.

To define a place, use the @ TYPE NAME prefix operation.

A key tool which should be used to the biggest extent is the macro operation.

The macro operator is denoted like this: = PATTERN EXPR where the pattern is a special kind of token group.

Inside the pattern, there can be a multitude of "arguments" denoted as $ NAME they serve as a generic slot for expressions to be put into them.

Inside of the macro expression (EXPR), there can be references to the names of the arguments in the pattern. When the macro is used, the expressions which are in place of the arguments will be put in place of the references.

Examples:

  • Simple constant macro:
    • = FLOAT_BITS 32;
  • Incrementer:
    • = (inc $A) ($A++);
  • C-like if-else statement definition:
    • = (
	if($condition)
	{
		$on_true
	}
	else
	{
		$on_false
	}
)
(
	$condition ? $on_true : $on_false
)
  • Minified C-like if-else statement definition:
    • =(if($A){$B}else{$C})($A?$B:$C)
